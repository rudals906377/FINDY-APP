# FINDY QA 체크리스트

이 문서는 기능을 추가하거나 화면을 수정한 뒤 수동으로 확인해야 할 기준입니다. 자동 검증은 `smoke_test.py`가 담당하고, 실제 화면 품질은 아래 항목으로 확인합니다.

## 1. 공통 화면 기준

- 모든 주요 화면의 뒤로가기 버튼이 바로 이전 화면으로 복귀합니다.
- 버튼, 칩, 날짜, 토글을 눌러도 화면이 맨 위로 튀지 않습니다.
- 접기/펼치기, 필터 선택, 날짜 선택 시 전체 화면이 깜빡이지 않고 해당 카드나 영역만 자연스럽게 갱신됩니다.
- 카드, 입력창, 칩은 기본적으로 흰색 배경과 `#E6D7C8` 라인을 사용합니다.
- 선택된 탭, 주요 버튼, 강조 아이콘만 `#8B6B4F`를 사용합니다.
- 텍스트가 카드, 버튼, 칩 밖으로 잘리지 않습니다.
- 그래프, 슬라이드 카드, 하단 네비게이션이 화면 밖으로 잘리지 않습니다.
- 가로 스크롤바가 보이지 않아야 합니다. 칩이 가로 스크롤되는 화면도 스크롤바는 숨깁니다.
- 로고는 `assets/app_logo`의 이미지 파일만 사용합니다. 텍스트로 로고를 직접 만들지 않습니다.

## 2. 고객 모드 필수 흐름

- `python3 FINDY_customer.py`로 고객 모드가 실행됩니다.
- 첫 진입은 고객 로그인 화면 기준으로 동작합니다.
- 네이버, 카카오, 구글, 애플 로그인 버튼이 홈 화면으로 이동합니다.
- 회원가입 화면은 고객 회원가입으로 열리고, 하단에서 아티스트 회원가입으로 이동할 수 있습니다.
- 홈 화면에서 검색창, 카테고리, 오늘의 스냅, 실제 리뷰, 추천 아티스트가 정상 노출됩니다.
- 카테고리 6개는 기존 홈 카드 디자인을 유지합니다.
- 카테고리 화면에서 리뷰, 커뮤니티, 아티스트, 샵 전체보기가 각각 연결됩니다.
- 헤어, 네일아트, 메이크업, 반영구, 웨딩, 포토 필터가 동일 속성 페이지와 연동됩니다.
- 리뷰 작성은 별점 0.5 단위 선택, 샵 이름, 지역, 아티스트, 시술명 입력을 지원합니다.
- 스냅, 비디오, 커뮤니티 작성 화면은 사진 또는 영상 입력 상태를 자연스럽게 처리합니다.
- 좋아요, 저장, 댓글 버튼은 화면 이동 없이 상태만 바뀝니다.
- 내정보에서 내가 쓴 글, 저장한 뷰티, 고객센터, 로그아웃이 동작합니다.

## 3. 아티스트 모드 필수 흐름

- `python3 FINDY_artist.py`로 아티스트 모드가 실행됩니다.
- 아티스트 로그인 화면에서 네이버, 카카오, 구글, 애플 로그인 버튼이 아티스트 메인으로 이동합니다.
- 고객 모드로 전환하는 버튼은 없습니다. 아티스트 모드는 아티스트 화면만 사용합니다.
- 아티스트 모드에서는 리뷰 작성 기능을 제공하지 않습니다.
- 아티스트 홈에는 검색창, 최근 주목 받는 스타일, 스냅, 고객 리뷰가 노출됩니다.
- 최근 주목 받는 스타일 카드는 일별, 주별, 월별 필터가 동작합니다.
- 트렌드 분석 페이지에서 카테고리별 검색 흐름과 선 그래프가 잘리지 않습니다.
- 관리 탭의 프로필 관리 섹션, 예약 관리 섹션, 리뷰 관리 섹션, 내 활동, 고객 지원이 분리되어 보입니다.
- 프로필 완성도 카드는 맨 위에 있고 접기/펼치기가 화면 튐 없이 동작합니다.
- 프로필 완성도 항목을 누르면 해당 수정 화면으로 이동합니다.
- 프로필 주요 정보는 운영팀 컨펌 대상입니다. 이름, 분야, 지역, 샵 이름은 임의 수정이 아니라 안내 중심으로 처리합니다.
- 포트폴리오 관리는 가격 메뉴와 연결됩니다. 가격 메뉴를 먼저 만들고 포트폴리오 등록 시 선택합니다.
- 포트폴리오 추가 화면에서 뒤로가기 버튼이 정상 동작합니다.
- 예약 관리는 월간 달력을 먼저 보여주고, 날짜 선택 시 해당 날짜 일정만 부드럽게 갱신합니다.
- 예약 변경 요청은 문자 메시지처럼 고객에게 보낼 문구를 입력하는 구조입니다.
- 취소 요청은 취소 사유를 입력하는 구조입니다.
- 메모 버튼은 해당 예약에 대한 메모 입력/저장을 지원합니다.
- 로그아웃은 아티스트 로그인 화면으로 돌아갑니다.

## 4. 빈 상태와 오류 상태

- 작성한 글이 없을 때는 빈 상태 카드가 명확하게 보입니다.
- 저장한 뷰티가 없을 때는 저장 안내 문구가 보입니다.
- 아티스트 리뷰가 없을 때는 받은 리뷰가 없다는 안내가 보입니다.
- 포트폴리오가 없을 때는 포트폴리오 추가 버튼이 가장 먼저 보입니다.
- 가격 메뉴가 없을 때는 가격 메뉴를 먼저 등록하라는 안내가 보입니다.
- 예약이 없는 날짜를 누르면 예약 없음 상태가 보이고 화면이 튀지 않습니다.
- 이미지 파일이 선택되지 않은 상태에서도 화면이 깨지지 않습니다.
- 필수 입력값이 없는 등록 화면은 저장 전에 누락 항목을 알립니다.

## 5. 최종 수동 확인 순서

1. 고객 모드 실행
2. 로그인 → 홈 → 카테고리 → 상세 글 → 뒤로가기
3. 리뷰 작성 → 저장 → 내정보에서 확인
4. 스냅/비디오/커뮤니티 작성 진입 확인
5. 좋아요/저장/댓글 상태 변경 확인
6. 로그아웃 확인
7. 아티스트 모드 실행
8. 로그인 → 아티스트 메인 → 프로필 완성도 접기/펼치기
9. 프로필 관리, 포트폴리오 관리, 가격 메뉴 관리 진입 확인
10. 예약 관리 → 날짜 선택 → 변경 요청/취소 요청/메모 확인
11. 트렌드 분석 그래프와 필터 확인
12. 로그아웃 확인
