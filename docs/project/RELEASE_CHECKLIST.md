# FINDY 출시 준비 체크리스트

이 문서는 FINDY 앱을 실제 배포 가능한 상태로 올리기 전 확인해야 할 항목을 정리합니다.

## 1. 실행 진입점

- 고객 모드: `FINDY_customer.py`
- 아티스트 모드: `FINDY_artist.py`
- 공통 앱: `FINDY.py`
- 웹사이트: `web/index.html`

실제 연결 구조는 `md/APP_STRUCTURE.md`를 기준으로 확인합니다.

## 2. 기본 검증

수정 후 아래 명령을 통과해야 합니다.

```bash
PYTHONPYCACHEPREFIX=/private/tmp/findy_pycache python3 -m py_compile FINDY.py FINDY_customer.py FINDY_artist.py
PYTHONPYCACHEPREFIX=/private/tmp/findy_pycache python3 smoke_test.py
git diff --check
```

## 3. 고객 모드 필수 흐름

- 로그인 화면 진입
- 고객 회원가입 화면 진입
- 홈 화면 진입
- 검색, 카테고리, 스냅, 비디오, 내정보 이동
- 리뷰/스냅/비디오/커뮤니티 작성 진입
- 좋아요, 저장, 댓글 동작
- 저장한 뷰티와 내가 쓴 글 확인
- 로그아웃 후 로그인 화면 복귀

## 4. 아티스트 모드 필수 흐름

- 아티스트 로그인 화면 진입
- 아티스트 메인 진입
- 프로필 완성도 접기/펼치기
- 프로필 관리 진입
- 포트폴리오 관리 진입
- 가격 메뉴 관리 진입
- 포트폴리오 추가 시 가격 메뉴 선택
- 예약 관리 월간 달력 진입
- 예약 변경 요청, 취소 요청, 메모 작성
- 리뷰 관리 진입
- 내 활동과 고객센터 진입
- 로그아웃 후 로그인 화면 복귀

## 5. 데이터 구조 기준

- 아티스트 프로필은 기본값과 정규화 함수를 통해 항상 같은 키를 유지합니다.
- 가격 메뉴는 포트폴리오보다 먼저 존재해야 합니다.
- 포트폴리오는 `menu_id`로 가격 메뉴와 연결합니다.
- 기존 더미 데이터는 화면 진입 시 현재 구조로 보정됩니다.
- 고객 문의와 예약은 분리합니다.
- 아티스트 주요 정보는 운영팀 컨펌 후 반영합니다.
- 세부 데이터 기준은 `md/DATA_STRUCTURE.md`를 따릅니다.

## 6. 디자인 기준

- 대표색, 로고색, 선택 상태, 주요 버튼: `#8B6B4F`
- 카드, 입력창, 칩 기본 배경: `#FFFFFF`
- 카드 테두리, 입력창 테두리, 구분선: `#E6D7C8`
- 선택된 칩과 주요 액션만 포인트 컬러를 사용합니다.
- 로고는 `assets/app_logo` 이미지 파일만 사용합니다.

## 7. UX 회귀 체크

- 뒤로가기 버튼은 모든 신규 페이지에서 바로 이전 화면으로 이동합니다.
- 토글, 필터, 날짜, 액션 버튼 클릭 시 화면이 맨 위로 튀지 않습니다.
- 접기/펼치기는 해당 카드만 부드럽게 변하고 전체 화면이 깜빡이지 않습니다.
- 그래프와 카드 내용은 좌우 또는 하단으로 잘리지 않습니다.
- 칩 목록은 가로 스크롤이 필요해도 스크롤바가 노출되지 않습니다.
- 하단 네비게이션은 고객 모드와 아티스트 모드에서 동일한 높이와 선택 스타일을 유지합니다.
- 화면 안정성 세부 기준은 `md/UI_STABILITY_GUIDE.md`를 따릅니다.

## 8. 출시 전 남은 결정 사항

- 실제 소셜 로그인 API 연결
- 실제 예약 데이터 저장소 연결
- 아티스트 컨펌/승인 운영툴
- 이미지 업로드 저장소
- 신고/숨김 처리 정책
- 개인정보처리방침, 이용약관
- 배포 대상 플랫폼과 빌드 방식
