# FINDY 데이터 구조 기준

이 문서는 현재 프로토타입 데이터와 향후 실제 서비스 데이터 구조를 정리한 기준입니다.

## 1. 현재 프로토타입 데이터 소스

```text
data/artists.py
data/categories.py
data/beauty_mood_keywords.py
data/reviews.py
data/snaps.py
reservation_history.json
app_state
```

- `data/artists.py`: 아티스트 카드, 추천 아티스트, 아티스트 프로필 화면의 기본 데이터입니다.
- `data/categories.py`: 헤어, 네일아트, 메이크업, 반영구, 웨딩, 포토 카테고리 기준입니다.
- `data/beauty_mood_keywords.py`: FINDY2 추천 알고리즘 준비용 카테고리별 감성어, 색감, 형태, 고민, 목적 키워드 사전입니다.
- `data/reviews.py`: 리뷰 목록과 리뷰 상세 화면의 기본 데이터입니다.
- `data/snaps.py`: 스냅 목록과 스냅 상세 화면의 기본 데이터입니다.
- `reservation_history.json`: 예약 기록 프로토타입 데이터입니다.
- `app_state`: 현재 화면 상태, 선택 탭, 임시 입력값, 토글 상태 등 런타임 UI 상태입니다.

## 2. 핵심 엔티티

향후 실제 데이터베이스로 옮길 때는 아래 단위로 분리합니다.

```text
Customer
ArtistProfile
ArtistApproval
PriceMenu
PortfolioItem
Reservation
Review
ContentPost
MessageThread
Notification
SupportTicket
FindyPreference
```

## 3. 주요 관계

- `ArtistProfile`은 아티스트 이름, 분야, 지역, 샵 이름, 소개, 링크, 영업시간, 상담 가능 여부를 가집니다.
- 아티스트 주요 정보인 이름, 분야, 지역, 샵 이름은 운영팀 컨펌 이후 반영합니다.
- `ArtistApproval`은 명함, 인스타그램, 네이버 플레이스 등록 사진 확인 상태를 관리합니다.
- `PriceMenu`는 시술명, 카테고리, 소요 시간, 가격, 설명, 예약 가능 여부를 가집니다.
- `PortfolioItem`은 반드시 하나 이상의 가격 메뉴와 연결됩니다.
- 고객 문의와 예약은 분리합니다. 문의는 메시지 흐름이고, 예약은 날짜/시간 체결 흐름입니다.
- 고객은 아티스트에게 먼저 메시지를 보낼 수 있고, 아티스트는 답장 중심으로 사용합니다.
- 아티스트 모드에서는 리뷰 작성 기능을 제공하지 않습니다.
- FINDY2는 고객/아티스트 구분 없이 `ContentPost`, `Review`, `FindyPreference` 중심으로 데이터를 수집합니다.

## 4. 프로토타입 저장 기준

- 화면 표시용 더미 데이터는 `data/`에 둡니다.
- 사용자가 입력한 프로토타입 기록은 JSON 또는 명확한 데이터 모듈에 둡니다.
- 단순 UI 상태는 `app_state`에 둡니다.
- 화면 토글, 필터, 선택 날짜처럼 영구 저장이 필요 없는 값은 실제 데이터와 섞지 않습니다.

## 5. 가격 메뉴와 포트폴리오 기준

포트폴리오와 가격 메뉴는 같은 개념이 아니라 연결된 개념입니다.

```text
가격 메뉴 생성 → 포트폴리오 등록 시 가격 메뉴 선택 → 고객 화면에서 가격/포트폴리오 함께 노출
```

- 가격 메뉴를 먼저 설정합니다.
- 포트폴리오 추가 화면에서 등록된 가격 메뉴를 선택합니다.
- 포트폴리오가 삭제되어도 가격 메뉴는 별도로 남을 수 있습니다.
- 가격 메뉴가 삭제되면 연결된 포트폴리오에는 “메뉴 확인 필요” 상태를 표시합니다.

## 6. 예약 데이터 기준

- 고객이 예약하면 기본 상태는 즉시 체결입니다.
- 아티스트는 예약 변경 요청과 취소 요청을 고객에게 메시지처럼 보냅니다.
- 예약 변경 요청은 날짜, 시간 후보, 사유/메시지를 포함합니다.
- 취소 요청은 취소 사유/메시지를 포함합니다.
- 메모는 아티스트 내부 기록이며 고객에게 노출하지 않습니다.

## 7. 메시지 데이터 기준

- 고객이 먼저 아티스트에게 문의합니다.
- 아티스트는 문의함에서 답장합니다.
- 문의와 예약은 분리합니다.
- 문의가 예약으로 자동 전환되지 않습니다.
- 추후 예약 연결이 필요하면 메시지 안에 “예약하기” 액션을 별도 버튼으로 추가합니다.

## 8. FINDY2 추천 정보 기준

- `FindyPreference`는 헤어, 네일아트, 메이크업, 포토, 웨딩, 반영구시술 카테고리별로 분리합니다.
- 카테고리마다 스타일, 고민/특징, 목적, 예산, 중요 조건, 메모를 따로 관리합니다.
- 감성어/색감/형태/기법/고민/목적 키워드는 `data/beauty_mood_keywords.py`와 `docs/project/BEAUTY_KEYWORD_TAXONOMY.md`를 기준으로 관리합니다.
- 사용자가 입력한 문장형 취향은 카테고리, 분위기, 색상, 형태, 목적 태그로 나눠 저장할 수 있게 설계합니다.
- FINDY2에서 쌓은 선호 데이터는 추후 FINDY 추천 알고리즘과 예약 추천 품질을 높이는 기준 데이터로 사용합니다.
