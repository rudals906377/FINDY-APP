# FINDY2 휴대폰 인증과 계정 복구 설계

기준일: 2026년 6월 25일

## 적용 원칙

FINDY2는 휴대폰 인증을 두 단계로 구분합니다.

| 방식 | 확인 범위 | FINDY2 사용처 |
|---|---|---|
| 문자 인증번호(SMS OTP) | 해당 휴대폰을 현재 소지하고 있는지 확인 | 회원가입, 아이디 찾기, 비밀번호 재설정 |
| PASS 휴대폰 본인확인 | 통신사 가입 명의와 본인 여부 확인 | 중복 가입 방지, 고위험 계정 복구, 향후 결제·예약 |

문자 인증은 실명이나 휴대폰 명의자를 확인하는 수단으로 표시하지 않습니다. 기존 비밀번호는 어떤 경우에도 문자나 화면으로 제공하지 않고, 인증 완료 후 새 비밀번호로 교체합니다.

## 현재 구현

- `POST /v1/phone/otp/request`: 6자리 인증번호 발급
- `POST /v1/phone/otp/verify`: 인증번호 검증
- 인증번호 유효 시간 3분
- 동일 번호와 목적의 재발송 제한 60초
- 최대 입력 시도 5회
- 서버에는 평문 인증번호 대신 HMAC 다이제스트 저장
- 회원가입 시 인증된 번호 중복 사용 차단
- 아이디 찾기 결과의 이메일 마스킹
- 비밀번호 재설정 시 영문과 숫자를 포함한 8자 이상 적용
- `POST /v1/identity/start`: PASS 본인확인 공급자 연결을 위한 서버 경계

개발 환경의 `FINDY_SMS_PROVIDER=console`은 인증번호를 서버 콘솔에 표시합니다. 운영 환경에서는 사용할 수 없으며 문자 공급자 Webhook을 연결해야 합니다.

## 한국에서 선택할 수 있는 본인확인 방식

1. PASS 앱 본인확인
   - NICE, KCB, 다날, KCP 등 본인확인기관과 계약
   - 통신 3사 인증, 앱 PIN·생체 인증 또는 문자 일반 인증 제공
   - CI/DI는 서버에서만 처리하고 앱이나 로그에 노출하지 않음

2. 문자 인증번호
   - NHN Cloud Notification, Solapi, 알리고 등 문자 발송 사업자 사용 가능
   - 번호 소지 확인에는 적합하지만 실명 확인이나 명의 확인으로 사용하면 안 됨

3. 아이핀 또는 공동·금융 인증서
   - 본인 명의 휴대폰 사용이 어려운 이용자의 대체 수단
   - FINDY2 초기 버전에는 필수 범위가 아니며 이용자 요구가 확인되면 추가

4. 카드 본인확인
   - 일부 본인확인기관에서 제공
   - 결제·예약 기능이 포함되는 FINDY 정식 버전에서 검토

## 운영 연결 전 남은 작업

- 본인확인기관 및 문자 발송 사업자 계약
- 운영 도메인과 HTTPS 콜백 등록
- PASS 성공·실패·취소 콜백 어댑터 구현
- 문자 Webhook을 실제 공급자 API로 교체
- OTP 요청 제한을 메모리에서 Redis 등 공유 저장소로 이전
- 휴대폰 번호 암호화 저장과 키 관리
- CI/DI의 별도 암호화·접근통제·보유기간 확정
- 개인정보 처리방침과 스토어 데이터 공개 최종 갱신
