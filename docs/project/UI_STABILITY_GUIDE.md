# FINDY UI 안정성 기준

이 문서는 버튼 클릭, 토글, 필터, 날짜 선택 시 화면이 튀거나 깜빡이는 문제를 줄이기 위한 기준입니다.

## 1. 화면 튐 방지 원칙

필터, 칩, 날짜, 토글, 접기/펼치기는 페이지 이동이 아닙니다. 이런 동작은 전체 화면을 다시 그리지 말고 해당 카드나 영역만 갱신합니다.

권장 기준:

```text
페이지 이동: 전체 렌더링 가능
현재 카드 상태 변경: 카드 내부 controls만 갱신
날짜/필터 변경: 목록 영역만 갱신
토글 변경: 토글과 관련 문구만 갱신
```

회귀로 보는 상황:

- 클릭 후 화면이 맨 위로 이동합니다.
- 화면 전체가 흰색으로 깜빡입니다.
- 하단 네비게이션이 다시 그려지는 느낌이 납니다.
- 스크롤 위치가 의도 없이 초기화됩니다.

## 2. 버튼 안정성 기준

모든 버튼은 `on_click`에서 아래 중 하나를 수행해야 합니다.

```text
1. navigate 계열 함수 호출
2. 특정 컨테이너 controls 갱신
3. 다이얼로그/오버레이 열기
4. 토스트 또는 상태 문구 표시
```

`page.controls.clear()`나 전체 페이지 재구성은 실제 화면 이동일 때만 사용합니다.

## 3. 뒤로가기 기준

- 모든 신규 페이지는 뒤로가기 버튼을 제공합니다.
- 뒤로가기는 바로 이전 화면으로 복귀합니다.
- 이전 화면 기록이 없을 경우 고객 모드는 홈, 아티스트 모드는 아티스트 메인으로 돌아갑니다.
- 뒤로가기 후 선택된 하단 탭이 현재 화면과 맞아야 합니다.

## 4. 모바일 비율 기준

- 본문 최대 폭은 앱 기준 폭 안에서 유지합니다.
- 카드 내부 좌우 패딩은 충분히 두어 글자와 그래프가 잘리지 않게 합니다.
- 긴 버튼 문구는 고정 폭보다 반응형 폭을 우선합니다.
- 하단 네비게이션과 콘텐츠가 겹치지 않도록 하단 여백을 유지합니다.
- 가로 칩 목록은 스크롤 가능하더라도 스크롤바는 숨깁니다.

## 5. 그래프 기준

- 그래프는 좌우 여백을 충분히 확보합니다.
- 첫 번째/마지막 점과 라벨이 카드 밖으로 나가지 않아야 합니다.
- 긴 카테고리명은 두 줄까지 허용하되, 다음 요소와 겹치면 안 됩니다.
- 점, 선, 라벨은 `#8B6B4F` 중심으로 통일합니다.

## 6. 색상 기준

- 대표색, 선택 상태, 주요 버튼: `#8B6B4F`
- 카드/입력창/칩 기본 배경: `#FFFFFF`
- 카드 테두리, 입력창 테두리, 구분선: `#E6D7C8`
- 선택되지 않은 칩은 흰색 배경과 라인만 사용합니다.
- 연한 베이지 배경은 꼭 필요한 보조 영역에만 제한적으로 사용합니다.

## 7. 신규 화면 추가 전 체크

신규 화면 또는 신규 카드 추가 전 아래를 확인합니다.

```text
- 뒤로가기 연결 여부
- 하단 네비게이션 선택 상태
- 버튼 무반응 여부
- 스크롤 위치 초기화 여부
- 모바일 폭에서 텍스트 잘림 여부
- 빈 상태와 완료 상태 존재 여부
```
