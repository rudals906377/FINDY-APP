# FINDY 웹사이트 제작 보고서

## 1. 주제

이번 홈페이지 제작 과제의 주제는 **FINDY - 리뷰와 커뮤니티를 기반으로 뷰티 경험을 찾는 웹사이트**이다.

FINDY는 "Find Your Beauty"라는 문장을 핵심 슬로건으로 삼은 웹사이트이다. 사용자가 헤어, 네일아트, 메이크업, 반영구, 웨딩, 포토 등 다양한 뷰티 분야에서 실제 리뷰와 커뮤니티 글을 확인하고, 자신에게 맞는 뷰티 스타일과 정보를 찾아볼 수 있도록 구성하였다.

웹사이트는 단순히 정보를 나열하는 소개 페이지가 아니라, 사용자가 직접 리뷰를 작성하고 커뮤니티 글을 남기며, 좋아요와 저장 기능을 통해 관심 있는 글을 모아볼 수 있는 형태로 제작하였다. 또한 로그인, 내정보, 나의 뷰티 작성, 리뷰 상세 보기, 커뮤니티 상세 보기 등 실제 서비스처럼 사용할 수 있는 기능을 포함하였다.

---

## 2. 주제 선택 배경

뷰티 서비스를 선택할 때 사용자는 보통 SNS, 검색 포털, 지도 앱, 후기 플랫폼 등을 여러 번 이동하며 정보를 찾는다. 하지만 이런 방식은 시간이 오래 걸리고, 광고성 후기와 실제 후기를 구분하기 어렵다는 문제가 있다.

특히 헤어, 네일아트, 메이크업처럼 결과물이 개인의 취향에 크게 영향을 받는 분야에서는 단순한 가격이나 별점만으로 선택하기 어렵다. 사용자는 자신이 원하는 분위기, 스타일, 감성에 맞는 아티스트나 샵을 찾고 싶어 하지만, 기존 검색 방식에서는 원하는 정보를 한눈에 비교하기 어렵다.

이러한 문제를 해결하기 위해 FINDY 웹사이트를 기획하였다. FINDY는 사용자가 실제 리뷰와 커뮤니티 글을 통해 뷰티 선택에 필요한 정보를 얻고, 자신이 추구하는 스타일을 기록하여 더 나에게 맞는 뷰티 경험을 찾는 것을 목표로 한다.

이 주제를 선택한 이유는 다음과 같다.

1. 실제 생활에서 자주 이용하는 뷰티 서비스와 관련된 주제이기 때문이다.
2. 리뷰와 커뮤니티 기능을 통해 사용자 참여형 웹사이트를 만들 수 있기 때문이다.
3. HTML5, CSS3, JavaScript를 활용하여 다양한 화면 전환과 기능을 구현하기 좋기 때문이다.
4. 단순한 정적 페이지가 아니라 로그인, 글쓰기, 저장, 좋아요 등 웹 서비스의 기본 흐름을 표현할 수 있기 때문이다.

## 3. 사용된 주요 기술

### 3.1 HTML5

HTML5는 웹사이트의 전체 구조를 만들기 위해 사용하였다.  
각 화면은 `section` 태그를 중심으로 구성하였고, 화면마다 `data-view` 속성을 부여하여 JavaScript로 화면 전환이 가능하도록 만들었다.

주요 HTML 구조는 다음과 같다.

- 헤더 영역
- 로그인 화면
- 홈 화면
- 리뷰 작성 화면
- 리뷰 목록 화면
- 리뷰 상세 화면
- 커뮤니티 목록 화면
- 커뮤니티 상세 화면
- 커뮤니티 글쓰기 화면
- 내정보 화면
- 나의 뷰티 작성 화면
- 앱 소개 및 출시 알림 화면
- 푸터 영역

HTML5의 시맨틱 구조를 활용하여 화면을 기능별로 나누었으며, 사용자가 클릭하거나 입력하는 부분은 `button`, `input`, `textarea`, `select`, `form` 태그를 사용하여 제작하였다.

### 3.2 CSS3

CSS3는 웹사이트의 디자인, 레이아웃, 반응형 화면, 카드 스타일, 버튼 스타일, 애니메이션 효과를 구현하기 위해 사용하였다.

사용한 CSS3 주요 기능은 다음과 같다.

1. **Flexbox**

   헤더 버튼, 카드 내부 요소, 액션 버튼, 로그인 버튼 목록 등을 정렬하기 위해 사용하였다.

2. **CSS Grid**

   홈 화면 카테고리 카드, 리뷰 목록, 내정보 페이지의 프로필과 콘텐츠 영역, 작성 폼의 2열 구조 등을 만들기 위해 사용하였다.

3. **border / border-radius**

   카드형 UI와 버튼을 부드럽고 고급스럽게 표현하기 위해 사용하였다. FINDY 웹사이트는 전체적으로 둥근 카드와 얇은 테두리를 사용하여 미니멀한 분위기를 만들었다.

4. **box-shadow**

   카드가 배경에서 자연스럽게 떠 보이도록 그림자 효과를 적용하였다. 너무 강한 그림자는 사용하지 않고, 은은한 그림자를 사용하여 웹사이트의 고급스러운 느낌을 유지하였다.

5. **transition**

   버튼에 마우스를 올렸을 때 색상이 바뀌거나 살짝 이동하는 효과를 주었다. 이를 통해 사용자가 클릭 가능한 요소를 쉽게 인식할 수 있도록 하였다.

6. **media query**

   화면 너비가 줄어들었을 때 카드가 한 줄로 쌓이거나, 사이드바가 하단 가로 메뉴처럼 보이도록 반응형 처리를 하였다.

### 3.3 JavaScript

JavaScript는 웹사이트의 동작을 구현하기 위해 사용하였다.  
FINDY 웹사이트는 별도의 서버 없이 브라우저에서 바로 실행되는 정적 웹사이트이기 때문에, JavaScript를 사용하여 화면 전환과 데이터 저장 기능을 구현하였다.

주요 JavaScript 기능은 다음과 같다.

1. **화면 전환 기능**

   `showView()` 함수를 사용하여 홈, 리뷰, 커뮤니티, 로그인, 내정보 등 여러 화면을 전환할 수 있게 만들었다.  
   또한 브라우저의 뒤로가기 기능도 사용할 수 있도록 `history.pushState()`와 `popstate` 이벤트를 적용하였다.

2. **로그인 기능**

   네이버, 카카오, 구글, 애플 로그인 버튼을 만들었다. 실제 외부 로그인 API를 연결한 것은 아니지만, 과제용 데모로 버튼을 누르면 로그인 상태가 저장되도록 구현하였다.

3. **리뷰 작성 기능**

   사용자가 닉네임, 카테고리, 샵 이름, 지역, 아티스트, 시술명, 별점, 리뷰 내용, 사진을 입력하여 리뷰를 작성할 수 있도록 만들었다.

4. **커뮤니티 글쓰기 기능**

   사용자가 카테고리와 글 종류를 선택하고, 제목과 내용을 입력하여 커뮤니티 글을 작성할 수 있도록 구현하였다.

5. **사진 업로드 및 미리보기**

   리뷰와 커뮤니티 글 작성 시 사진을 첨부할 수 있게 하였고, 첨부한 사진은 미리보기로 확인할 수 있도록 만들었다.

6. **좋아요, 저장, 댓글 버튼**

   리뷰와 커뮤니티 글에 좋아요, 저장, 댓글 버튼을 추가하였다. 좋아요와 저장은 클릭 시 상태가 바뀌고, 내정보 페이지에서 확인할 수 있도록 하였다. 댓글 기능은 현재 준비 중 메시지가 나타나도록 처리하였다.

7. **localStorage 저장**

   서버나 데이터베이스 없이 브라우저의 `localStorage`를 사용하여 리뷰, 커뮤니티 글, 로그인 상태, 좋아요, 저장, 나의 뷰티 정보를 저장하였다.

## 4. 웹사이트 주요 기능 설명

### 4.1 홈 화면

홈 화면은 FINDY 웹사이트의 첫 화면이다.  
상단에는 FINDY 로고와 로그인 버튼, 리뷰 작성 버튼, 커뮤니티 참여 버튼을 배치하였다.

중앙에는 "Find Your ..." 형태의 문구가 나타나며, 헤어, 네일아트, 메이크업, 반영구, 웨딩, 포토, Beauty 등의 단어가 자동으로 전환된다. 이를 통해 FINDY의 핵심 슬로건인 "Find Your Beauty"를 강조하였다.

홈 화면에는 검색창과 카테고리 카드도 배치하였다. 사용자는 원하는 키워드를 검색하거나 카테고리 카드를 눌러 리뷰와 커뮤니티 정보를 탐색할 수 있다.

### 4.2 로그인 화면

로그인 화면은 웹사이트 첫 진입 시 바로 나타나지 않도록 하였다.  
사용자가 글을 작성하거나 전체글을 보려고 할 때 로그인하지 않은 상태라면 로그인 화면으로 이동하도록 만들었다.

로그인 화면에는 FINDY 로고와 간단한 안내 문구, 네이버, 카카오, 구글, 애플 로그인 버튼을 배치하였다. 실제 서비스에서는 각 로그인 API를 연결해야 하지만, 현재 과제용 웹사이트에서는 버튼 클릭만으로 로그인 상태가 적용되도록 구현하였다.

### 4.3 리뷰 목록 화면

리뷰 목록 화면에서는 사용자가 작성한 리뷰와 기본 예시 리뷰를 카드 형태로 볼 수 있다.  
각 리뷰 카드에는 작성자, 카테고리, 시술명, 별점, 샵 이름, 지역, 아티스트, 리뷰 내용, 좋아요, 저장, 댓글 버튼이 표시된다.

왼쪽 사이드바에서 헤어, 네일아트, 메이크업, 반영구, 웨딩, 포토 카테고리를 선택하면 해당 카테고리의 리뷰만 볼 수 있도록 하였다.

### 4.4 리뷰 상세 화면

리뷰 카드를 클릭하면 해당 리뷰를 전체 화면으로 크게 볼 수 있다.  
왼쪽에는 선택한 리뷰의 전체 내용이 표시되고, 오른쪽에는 같은 카테고리의 리뷰 리스트가 표시된다.

이를 통해 사용자는 리뷰 목록 화면에서 간단히 내용을 확인하고, 관심 있는 리뷰를 클릭하여 더 자세히 볼 수 있다.

### 4.5 리뷰 작성 화면

리뷰 작성 화면에서는 사용자가 직접 리뷰를 등록할 수 있다.  
입력 항목은 다음과 같다.

- 닉네임
- 카테고리
- 샵 이름
- 지역
- 아티스트
- 시술명
- 별점
- 리뷰 내용
- 사진 첨부

별점은 0.5점 단위까지 선택할 수 있도록 구현하였다. 사진은 최대 4장까지 첨부할 수 있으며, 첨부한 사진은 미리보기로 확인할 수 있다.

### 4.6 커뮤니티 화면

커뮤니티 화면은 리뷰만으로 부족한 질문이나 추천 요청을 남길 수 있는 공간이다.  
사용자는 추천 요청, 질문, 전후 공유, 가격 공유와 같은 글 종류를 선택해 커뮤니티 글을 볼 수 있다.

커뮤니티 글에는 제목, 작성자, 카테고리, 글 종류, 내용, 좋아요, 저장, 댓글 버튼이 포함된다.

### 4.7 커뮤니티 상세 화면

커뮤니티 글을 클릭하면 전체글 보기 화면으로 이동한다.  
선택한 글을 크게 볼 수 있고, 오른쪽에는 전체 커뮤니티 글 리스트가 표시된다.

이 구조는 사용자가 한 글을 읽으면서 다른 글로 쉽게 이동할 수 있도록 하기 위해 만들었다.

### 4.8 커뮤니티 글쓰기 화면

커뮤니티 글쓰기 화면에서는 사용자가 직접 글을 등록할 수 있다.  
입력 항목은 다음과 같다.

- 뷰티 카테고리
- 글 종류
- 제목
- 내용
- 사진 첨부

사진을 첨부한 글은 목록과 상세 화면에서 작은 썸네일 형태로 표시되도록 하였다.

### 4.9 내정보 화면

로그인 후 상단의 "내정보" 버튼을 누르면 내정보 페이지로 이동한다.  
내정보 페이지는 웹사이트 느낌의 대시보드 형태로 구성하였다.

내정보 화면에는 다음 항목이 있다.

- 프로필
- 이름
- 성별
- 나이
- 지역
- 나의 뷰티
- 내가 작성한 리뷰
- 내가 작성한 커뮤니티
- 좋아요한 글
- 저장한 글
- 공지사항
- 1:1 문의

내가 작성한 리뷰나 커뮤니티 글을 클릭하면 해당 글의 전체글 보기 화면으로 이동한다.  
좋아요하거나 저장한 글도 내정보에서 확인할 수 있도록 만들었다.

### 4.10 나의 뷰티 작성 화면

나의 뷰티 작성 화면은 사용자가 자신이 추구하는 뷰티 스타일을 기록하는 공간이다.

입력 항목은 다음과 같다.

- 본인이 추구하는 느낌
- 선호하는 스타일
- 좋아하는 감성

저장한 내용은 내정보 페이지의 "나의 뷰티" 영역에 요약되어 표시된다.

## 5. 스토리보드

아래는 FINDY 웹사이트의 전체 흐름을 정리한 스토리보드이다.

### 5.1 기본 탐색 흐름

1. 사용자가 웹사이트에 접속한다.
2. 홈 화면에서 FINDY 로고, 검색창, 카테고리 카드를 확인한다.
3. 사용자는 원하는 카테고리를 선택하거나 검색어를 입력한다.
4. 선택한 카테고리에 맞는 리뷰 목록이 표시된다.
5. 리뷰 카드를 클릭하면 로그인 화면이 나타난다.
6. 로그인 후 선택한 리뷰의 상세 화면으로 이동한다.
7. 상세 화면에서 좋아요, 저장, 댓글 버튼을 확인할 수 있다.

### 5.2 리뷰 작성 흐름

1. 사용자가 상단의 "리뷰 작성" 버튼을 클릭한다.
2. 로그인하지 않은 상태라면 로그인 화면으로 이동한다.
3. 로그인 후 리뷰 작성 화면으로 이동한다.
4. 닉네임, 카테고리, 샵 이름, 지역, 아티스트, 시술명, 별점, 리뷰 내용을 입력한다.
5. 필요한 경우 사진을 첨부한다.
6. "리뷰 등록하기" 버튼을 클릭한다.
7. 작성한 리뷰가 리뷰 목록과 내정보 페이지에 표시된다.

### 5.3 커뮤니티 작성 흐름

1. 사용자가 "커뮤니티 참여" 버튼을 클릭한다.
2. 커뮤니티 목록 화면으로 이동한다.
3. "글 작성" 버튼을 클릭한다.
4. 로그인하지 않은 상태라면 로그인 화면으로 이동한다.
5. 로그인 후 커뮤니티 글쓰기 화면으로 이동한다.
6. 카테고리, 글 종류, 제목, 내용, 사진을 입력한다.
7. 글을 등록하면 커뮤니티 목록과 내정보 페이지에 표시된다.

### 5.4 내정보 흐름

1. 사용자가 로그인한다.
2. 상단의 로그인 버튼이 "내정보"로 변경된다.
3. "내정보" 버튼을 클릭하면 내정보 페이지로 이동한다.
4. 프로필 정보와 작성한 리뷰, 작성한 커뮤니티, 좋아요한 글, 저장한 글을 확인한다.
5. "나의 뷰티"를 클릭하면 뷰티 취향 작성 화면으로 이동한다.
6. 본인이 추구하는 느낌, 스타일, 감성을 작성하고 저장한다.
7. 저장한 내용이 내정보 페이지에 요약되어 표시된다.

## 7. 완성된 홈페이지의 주요 페이지 캡처화면

완성된 웹사이트에서 캡처하면 좋은 주요 페이지는 다음과 같다.

### 7.1 홈 화면

홈 화면에는 FINDY 로고, 로그인 버튼, 리뷰 작성 버튼, 커뮤니티 참여 버튼, 검색창, 카테고리 카드가 표시된다.  
사용자는 이 화면에서 웹사이트의 주요 기능으로 이동할 수 있다.

### 7.2 로그인 화면

로그인 화면은 사용자가 전체글을 보거나 글을 작성하려고 할 때 나타난다.  
네이버, 카카오, 구글, 애플 로그인 버튼을 배치하여 실제 앱과 비슷한 로그인 흐름을 표현하였다.

### 7.3 리뷰 목록 화면

리뷰 목록 화면에서는 실제 사용자 후기처럼 보이는 리뷰 카드들이 표시된다.  
좋아요, 저장, 댓글 버튼을 통해 사용자가 글에 반응할 수 있도록 하였다.

### 7.4 리뷰 상세 화면

리뷰 상세 화면에서는 선택한 리뷰를 크게 확인할 수 있다.  
오른쪽에는 리뷰 리스트가 함께 표시되어 다른 리뷰로 쉽게 이동할 수 있다.

### 7.5 리뷰 작성 화면

리뷰 작성 화면에서는 사용자가 직접 리뷰를 작성할 수 있다.  
샵 이름, 지역, 아티스트, 시술명, 별점, 사진 첨부 기능을 포함하여 실제 서비스처럼 보이도록 구성하였다.

### 7.6 커뮤니티 화면

커뮤니티 화면에서는 추천 요청, 질문, 전후 공유, 가격 공유 등 리뷰만으로 부족한 이야기를 나눌 수 있다.

### 7.7 커뮤니티 상세 화면

커뮤니티 상세 화면에서는 선택한 글의 전체 내용을 볼 수 있고, 오른쪽 리스트를 통해 다른 글로 이동할 수 있다.

### 7.8 커뮤니티 글쓰기 화면

커뮤니티 글쓰기 화면에서는 제목, 내용, 카테고리, 글 종류, 사진을 입력하여 글을 작성할 수 있다.

### 7.9 내정보 화면

내정보 화면은 로그인한 사용자의 대시보드처럼 구성하였다.  
프로필, 작성한 리뷰, 작성한 커뮤니티, 좋아요한 글, 저장한 글, 공지사항, 1:1 문의를 확인할 수 있다.

### 7.10 나의 뷰티 작성 화면

나의 뷰티 작성 화면에서는 사용자가 본인이 추구하는 느낌, 스타일, 감성을 기록할 수 있다.

## 8. 제작 후 느낀 점

이번 홈페이지 제작을 통해 단순히 화면을 예쁘게 만드는 것뿐만 아니라, 사용자가 어떤 순서로 서비스를 이용하게 될지 생각하는 것이 중요하다는 것을 알게 되었다.

처음에는 홈 화면과 리뷰 목록 중심으로 제작했지만, 작업을 진행하면서 로그인, 글쓰기, 전체글 보기, 내정보, 좋아요, 저장 기능이 추가되었다. 이 과정에서 웹사이트는 여러 화면이 서로 연결되어야 자연스럽게 사용할 수 있다는 것을 배웠다.

또한 HTML5로 구조를 만들고, CSS3로 레이아웃과 디자인을 정리하며, JavaScript로 실제 동작을 구현하는 과정을 통해 웹사이트 제작의 기본 흐름을 이해할 수 있었다.

특히 `localStorage`를 사용하여 서버 없이도 사용자의 리뷰, 커뮤니티 글, 로그인 상태, 좋아요, 저장 정보를 저장할 수 있다는 점이 흥미로웠다. 실제 서비스로 발전시키기 위해서는 데이터베이스와 서버가 필요하지만, 과제용 웹사이트에서는 충분히 웹 서비스의 흐름을 표현할 수 있었다.

FINDY 웹사이트는 뷰티 서비스를 선택할 때 필요한 리뷰와 커뮤니티 정보를 한곳에서 확인할 수 있도록 만든 사용자 참여형 웹사이트이다. 앞으로 더 발전시킨다면 실제 로그인 API, 댓글 기능, 사용자별 추천 기능, 예약 기능 등을 추가할 수 있을 것 같다.
