# FINDY2

FINDY2는 첫 배포용 커뮤니티형 앱 기준 파일입니다. 고객과 아티스트를 나누지 않고 모두가 사용할 수 있는 뷰티 커뮤니티 앱으로 운영하며, 여기에서 모은 데이터를 바탕으로 나중에 FINDY의 추천/예약 시스템을 강화합니다.

## 실행

```bash
cd /Users/kyoungmin/Desktop/FINDY
python3 python_files/FINDY2.py
```

실제 앱 진입점은 `python_files/FINDY2.py`이며, 상태·설정·기본 콘텐츠·저장소 로직은 역할별 모듈로 분리합니다.

배포 빌드는 프로젝트 루트의 `main.py`를 사용하며, `pyproject.toml`에 제품명, 번들 ID, 권한, 아이콘과 스플래시 기준을 관리합니다.

## 배포 기준

- 제품 버전: `0.1.0`
- 번들 ID 초안: `com.findybeauty.findy2`
- 앱 권한: 사진 보관함만 요청
- 앱 내 개인정보 처리방침, 이용약관, 커뮤니티 운영원칙 화면 제공
- 휴대폰 문자 인증 기반 회원가입, 아이디 찾기와 비밀번호 재설정
- PASS 휴대폰 본인확인 공급자 연동을 위한 인증 서버 경계
- 설정에서 비밀번호 재확인 후 계정과 로컬 사용자 데이터 삭제 가능
- 현재 베타 데이터는 기기 로컬에 저장되며 서버 연동 전 정책 재검토 필요
- 상세 빌드 절차: `docs/project/FINDY2_DEPLOYMENT.md`

## 현재 역할

- 첫 배포 기준 앱입니다.
- 예약 중심이 아니라 커뮤니티, 리뷰, 질문, 자유게시판, 스냅, 비디오 중심입니다.
- 아티스트 등록/운영 앱이 아니라 고객 중심 커뮤니티 앱입니다.
- 현재 FINDY2 앱 버전에서는 스냅/커뮤니티 작성 과정에 아티스트 선택이나 시술자 정보 노출을 두지 않습니다.
- FINDY와 디자인은 최대한 동일하게 유지하고, 기능만 커뮤니티 데이터 수집에 맞춥니다.

## 주요 기능

- 홈: 인기글, 리뷰, 질문, 자유게시판, FINDY 스냅, FINDY 비디오
- 하단 첫 탭: 카테고리 대신 커뮤니티로 운영
- 커뮤니티: 자유게시판, 리뷰, 질문만 한 화면에서 피드로 확인하고 스냅/비디오는 별도 하단 탭에서 운영
- 커뮤니티 글: 설정/내정보 화면과 같은 부드러운 라운드 카드, 연한 테두리, 태그형 배지 기준
- 커뮤니티/스냅 조작부: 작성, 분야, 정렬, 보기 전환처럼 같은 성격의 조작 정보는 한 줄 툴바로 압축
- 글 카드 하단 정보: 조회, 저장, 좋아요, 댓글은 본문과 얇은 구분선으로 분리하고 세로 여백을 최소화
- 글 상세 화면: 제목과 본문을 우선 배치하고 작성자, 좋아요, 저장, 신고는 작은 보조 동작으로 정리합니다.
- 종류별 상세 정보: 리뷰는 별점·시술 분야·인증 방문 여부, 질문은 답변 대기/완료 상태, 자유게시판은 간결한 일반 댓글 흐름을 제공합니다.
- 상세 댓글: 좋아요, 답글, 신고, 사진·장소·이모지 첨부와 전송을 지원하고 댓글 사진을 본문 아래에 표시합니다.
- 게시글 신고: 허위 정보, 욕설/비방, 개인정보 노출, 사생활 침해, 광고/스팸, 기타 사유와 상세 내용을 저장하고 검토 중 상태를 표시합니다.
- 글쓰기 화면: 커뮤니티, 스냅, 비디오 작성 페이지는 뒤로가기/완료 상단바, 한 줄 선택바, 큰 제목/내용 입력 영역, 하단 첨부 툴바 구조로 통일
- 공통 상단 액션바: FINDY 로고 옆에 현재 화면명(커뮤니티, 스냅, 비디오, 내정보 등)을 한 줄로 노출하고, 오른쪽에는 검색, 메시지 목록, 알림 버튼을 고정
- 검색 화면: 상단 돋보기 버튼을 누르면 전용 검색 레이어를 열고, 둥근 검색창, 사진 검색 준비 안내, 빠른 카테고리, FINDY 추천, 최근 검색어 삭제/재검색 동작을 제공
- 메인 탭 헤더: 본문 안 중복 헤더를 두지 않고 상단 공통바 하나를 기준점으로 사용
- 공통 레이아웃: 커뮤니티/내정보/검색/작성 등 일반 페이지는 좌우 여백을 줄여 콘텐츠 폭을 넓게 쓰고, 스냅/비디오 전용 화면은 기존 폭과 풀블리드 연출을 유지
- 화면 문구 정리: FINDY2 기본 동선에서는 아티스트/샵 중심 표현 대신 서비스, 방문 장소, 공개 표시명처럼 고객 커뮤니티에 맞는 중립 표현을 사용
- 버튼 피드백: 아직 연결 전인 기능은 `곧 연결될 예정` 톤으로 안내하고, 모든 클릭 피드백은 FINDY 메인 컬러 토스트로 통일
- 다중 선택 필터: 커뮤니티, 홈 카테고리 전용 페이지, 스냅, 비디오의 분야/정렬/보기 선택은 서랍형 선택창으로 제공하며 선택 상태를 유지
- 홈 카테고리 전용 페이지: 헤어, 네일아트, 메이크업, 포토, 웨딩, 반영구 분야와 전체, 리뷰, 질문, 자유게시판, 스냅, 비디오 콘텐츠를 서랍형으로 선택
- 홈 글 노출 순서: 인기글을 먼저 보여주고, 이후 리뷰, 질문, 자유게시판, 스냅, 비디오 순서로 고정
- 커뮤니티 글 정렬: 자유게시판, 리뷰, 질문 안에서 추천순, 최신순, 댓글순 중 사용자가 고른 정렬을 우선 적용
- 뷰티 분야: 헤어, 네일아트, 메이크업, 포토, 웨딩, 반영구시술 기준 유지
- 안전장치가 포함된 리뷰 작성/신고/상태 관리 로직
- 스냅/비디오 피드, 좋아요, 저장, 댓글, 공유, 신고, 팔로우 동작
- FINDY 비디오: 상단바 아래부터 여백 없이 풀블리드로 표시하고, 위/아래 슬라이드 시 다음/이전 영상 화면이 밀려 들어오는 전환감 적용
- 내정보: 내가 쓴글, 좋아요, 저장, 나의 FINDY, 설정, 프로필, 내 예약, 후기, 알림
- 내정보 서비스 연결: 내 예약은 예약내역, 후기는 내 후기, 알림은 공통 알림 페이지로 연결하며 쿠폰/이벤트는 준비 중 안내
- 나의 FINDY: 카테고리별 추천 정보 입력
- 화면 문구 안정화: 고객 커뮤니티 앱 기준에 맞춰 `나의 뷰티정보`, `시술`, `샵 정보`, `공개 프로필` 등 일부 구버전 표현을 `나의 FINDY`, `서비스/이용`, `방문 정보`, `공개 표시명` 중심으로 정리
- 기능 분리: FINDY2 현재 버전에서는 예약/직접 메시지를 비활성 플래그로 분리하고, 사용자 진입 시 준비 안내 화면으로 보냅니다. 예약/DM 코드는 추후 FINDY 본서비스에서 재활성화할 수 있도록 보관합니다.
- 로컬 사용자 데이터 저장: 작성글, 댓글, 좋아요, 저장, 리뷰/신고, 프로필, 나의 FINDY, 최근 검색어는 앱 재실행 후에도 유지합니다.
- 저장 안정성: 사용자 데이터는 버전이 포함된 JSON으로 원자적 저장하며, 기존 파일을 `.bak`으로 보관해 파일 손상 시 마지막 정상 데이터로 복구합니다.
- 계정 인증: 이메일, 사용자 이름, 닉네임, 비밀번호로 회원가입하고 이메일/비밀번호로 로그인합니다.
- 통합회원 로그인: 네이버, 카카오, Google, Apple 인증을 FINDY 인증 게이트웨이에서 처리하고 하나의 FINDY 사용자 ID로 연결합니다.
- 소셜 계정 병합: 공급자 사용자 ID, 인증 서버 사용자 ID, 검증 이메일 순서로 기존 계정을 찾아 글, 저장, 프로필 데이터가 분리되지 않도록 합니다.
- 연결된 계정 관리: 설정에서 네이버, 카카오, Google, Apple의 연결 상태와 인증 이메일을 확인하고 추가 로그인 수단을 연결할 수 있습니다.
- 연결 충돌 보호: 이미 다른 FINDY 계정에 연결된 소셜 계정은 현재 계정에 중복 연결할 수 없습니다.
- 비밀키 보호: 공급자 Client Secret과 Apple Private Key는 앱에 포함하지 않고 `auth_gateway/` 서버 환경 변수에서만 사용합니다.
- 인증 서버 배포: 전용 가상환경 실행 스크립트와 Docker Compose 구성을 제공하며 설정 검사기는 비밀값을 출력하지 않고 누락 여부만 확인합니다.
- 비밀번호 보호: 비밀번호 원문은 저장하지 않고 무작위 salt와 PBKDF2-SHA256 반복 해시만 로컬 회원 저장소에 보관합니다.
- 로그인 세션: 로그인 성공 시 무작위 세션 토큰을 별도 파일에 저장하며 앱 재실행 시 로그인 상태를 복원합니다. 로그아웃하면 세션 파일을 제거합니다.
- 작성자 권한: 새 커뮤니티 글, 리뷰, 댓글, 스냅, 비디오에 `userId`를 기록하고 본인 콘텐츠에만 삭제 동작을 제공합니다.
- 사진 업로드: 커뮤니티는 최대 6장, 리뷰/스냅은 최대 10장을 선택하고 작성 화면에서 미리보기와 개별 삭제를 제공합니다.
- 사진 최적화: 선택 즉시 형식·손상·20MB 제한을 검사하고 등록 시 EXIF 회전을 보정해 긴 변 2048px 이하 JPEG/WebP로 압축합니다.
- 비디오 업로드: MP4, MOV, M4V, WEBM 파일을 최대 250MB까지 검사하며 작성 화면에서 재생 미리보기와 선택 취소를 제공합니다.
- 미디어 저장 안정성: 게시 완료 시에만 FINDY2 전용 미디어 폴더로 복사하고 데이터 저장 실패 시 복사본을 자동 정리합니다.
- 계정별 데이터 분리: 사용자 생성 데이터는 회원 ID가 포함된 별도 파일에 저장해 다른 로컬 계정의 글, 좋아요, 프로필이 섞이지 않도록 합니다.
- 기존 데이터 이전: 이전 단일 사용자 파일이 있으면 첫 로그인 계정으로 한 번만 이전하고 원본은 `.migrated` 파일로 보관합니다.
- 화면 접근 보호: 로그인·회원가입·오프닝을 제외한 화면은 인증된 세션이 없으면 로그인 화면으로 이동합니다.
- 공통 오류 화면: 렌더링 오류가 발생하면 앱이 종료되지 않고 다시 시도 또는 홈 이동 화면을 제공합니다.
- 반응형 프레임: 창 크기 변경을 감지해 현재 페이지를 다시 배치하고, 좁은 화면에서는 상단 로고·제목·액션 버튼 크기와 간격을 줄입니다.
- 설정 저장: 알림, 소리, 개인정보, 자동 재생 등 설정 변경은 사용자별 데이터에 즉시 반영합니다.
- 자산 경로 검증: Flet의 `assets_dir` 기준으로 로고와 Pretendard 폰트를 참조해 `assets/assets` 중복 경로 오류를 방지합니다.
- 콘텐츠 필터 보정: 좋아요/저장 목록의 `자유게시판` 표기를 내부 `자유` 유형과 연결해 빈 목록으로 잘못 표시되지 않도록 합니다.
- 알림 분리: 현재 알림은 커뮤니티 활동, 리뷰, 나의 FINDY 안내 중심으로 구성하고 예약 알림은 예약 플래그가 켜졌을 때만 합쳐집니다.
- 소식함: 상단 종이비행기 아이콘은 외부 DM이 아니라 FINDY 안내, 비디오 팁, 커뮤니티 활동 알림을 확인하는 `소식함`으로 운영합니다.
- 추천 키워드 사전: 헤어, 네일아트, 메이크업, 포토, 웨딩, 반영구시술별 감성어/색감/기법/고민/목적 데이터

소셜 로그인 개발자 콘솔과 서버 설정은 `docs/project/SOCIAL_AUTH_SETUP.md`를 기준으로 합니다.

## 수정 기준

- 새 커뮤니티 기능은 우선 FINDY2에 구현합니다.
- FINDY2에서 안정화된 디자인과 흐름은 추후 FINDY에 반영합니다.
- 앱 안 표기명은 `FINDY2`를 사용합니다. 파일명도 `python_files/FINDY2.py`를 기준으로 합니다.
- 왁싱, 샵추천, 아티스트 추천은 FINDY2 기본 범위에 포함하지 않습니다.
- 메시지는 외부 DM/샵 상담 기능이 아니라 앱 내부 안내와 비디오 팁 공유용 목록으로만 운영합니다.
- 화면 코드는 당분간 `python_files/FINDY2.py`의 기존 흐름을 유지하되, 화면과 무관한 상태·설정·더미 데이터·저장 로직은 아래 역할별 모듈에서 관리합니다.

## 로컬 데이터 위치

개발 단계 사용자 데이터는 기본적으로 아래 위치에 저장합니다.

```text
~/.findy/FINDY2/user_data.json
~/.findy/FINDY2/user_data.json.bak
~/.findy/FINDY2/users.json
~/.findy/FINDY2/session.json
~/.findy/FINDY2/media/{userId}/images/
~/.findy/FINDY2/media/{userId}/videos/
```

테스트나 배포 환경에서는 `FINDY2_USER_DATA_PATH`, `FINDY2_AUTH_DATA_PATH`, `FINDY2_SESSION_PATH`, `FINDY2_MEDIA_DIR` 환경 변수로 저장 위치를 변경할 수 있습니다. 이후 Supabase를 연결할 때 화면 코드는 유지하고 이 저장·인증·미디어 계층을 원격 구현으로 교체합니다.
